c++ 필요한 개념 정리

c++의 '식' 대부분은 연산자와 피연산자를 조합하여 만든다.

컴파일 : c++을 기계어로 만드는 작업 / 파일러라는 소프트 웨어 사용

오브젝트 파일 : 컴파일이 완료되면 저장된 폴더( 디렉토리 )에, 기계어로 번역된 파일이 생성되는데
이 파일을 가리킴

링크(link) : 생성된 오브젝트 파일에 다른 프로그램과 함께 사용하는 오브젝트 파일을 상호 연결해, 실제로
실행 가능한 형식을 갖춘 프로그램을 하나 완성하도록 하는 작업 / 이를 수행하는 링커 ( LInker )

표준 출력 (stdard output) : 지금 사용하고 있는 컴퓨터 화면을 뜻하며 cout가 이에 해당

리다이렉트 : OS에 따라 표준 출력을 다른 장비로 바꿀 수 있는데 이러한 기능을 일컬어 표현하는 말이다.

 인클루드(include) : 파일을 읽는 작업 ( 개발환경에 표준으로 포함된 파일은 스스로 준비할 필요 x )

헤더파일 : 미리 읽어두는 파일 (iostream , stdio.h 가 이에 해당 )

리터럴( literal ) : 'A'등과 같은 특정 문자나 숫자 표기

변수 : 메모리를 이용해 값을 기억시켜 두는 기능 ( 식별자 : 변수의 이름으로 사용가능한 문자 / 숫지의 조합 )

변수 선언 과정

1. 변수의 이름과 형을 컴파일러에게 알림
2. 변수를 저장하기 위한 메모리 준비 ( 변수 정의라 부르기도 함 )

변수의 초기화 - 변수를 선언했을 떄, 동시에 변수에 값을 저장하는 처리 방법

const : 변수에 const가 지정되면, 그 변수의 값은 더 이상 바꿀 수 없게 됨.

사칙 연산에서는 우선 순위가 동일할 경우 반드시 ( 왼 -> 오 )로 계산하게 되어 있는데, 이를 왼쪽 결합이라 부름

캐스트 연산자 : 지정된 식의 형을 ( ) 안에서 지정된 형으로 변환시키는 연산자
- 연산자가 다른 두 피연산자의 사이즈를 비교해 그 중 큰 데이터 형으로 형 변환을 시킨 뒤 연산을 수행

함수 호출 : 함수를 정의한 뒤 다른 코드에서 활용할 수 았게 되어 이를 이용하는 행위
