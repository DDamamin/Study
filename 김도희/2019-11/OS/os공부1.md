 컴퓨터 하드웨어의 구성 


컴퓨터는 하드웨어와 소프트웨어로 구성이 되는데 운영체제는 하드웨어를 관리하는 소프트웨어 이다.

 

컴퓨터 하드웨어는 크게 프로세서 ( cpu ) , 메모리(기억장치) , 주변장치로 구성 되어 있다. (서로 시스템 버스로 연결 )

 

 

 

프로세서( cpu )

하드웨어에 부착한 모든 장치의 동작을 제어하고 명령을 실행

 

레지스터
레지스터는 컴퓨터의 프로세서 내에서 자료를 보관하는 아주 빠른 기억 장소

 

 

용도 : 전용 레지스터 / 범용레지스터

사용자가 정보를 입력할 수 있는지 : 사용자 가시 / 사용자 불가시 레지스터

정보의 종류 : 데이터 레지스터 , 주소 레지스터 , 상태 레지스터

 

 

 

사용자 가시 레지스터

 

데이터 레지스터 : 함수 연산에 필요한 데이터 저장

 

주소 레지스터 : 주소나 유효 주소 계산을 하는 데 필요한 주소의 읿부분을 저장

 

- 기준 주소 레지스터 : 프로그램 실행 시 사용하는 기준 주소값 저장

- 인덱스 레지스터 : 유효 주소를 계산하는데 사용하는 주소 정보 저장

- 스택 포인터 레지스터 : 메모리 프로세서 스텍을 구현할때 사용

 

 

사용자 불가시 레지스터

 

프로그램 카운터 ( PC ) : 다음에 실행할 명령어의 주소를 보관하는 레지스터

명령어 카운터 (  IR ) : 현재 실행하는 명령어를 보관 입력하는 레지스터

누산기 ( ACC ) : 데이터를 일시적 저장하는 레지스터

메모리 주소 레지스터 : 프로세스가 참조하려는 데이터의 주소를 명시해 메모리에 접근하는 버퍼 레지스터

메모리 버퍼 레지스터 : 프로세스가 메모리에서 읽거나 저장할 데이터를 보관하는 레지스터 

 

 

 

 
메모리
 

 

메모리 계층 구조

 

레지스터   

캐시                                ( 레지스터 캐시 메인 메모리 전부 프로세서가 프로그램과 데이터에 직접 접근 가능 )    

메인 메모리 

보조 기억 장치                   ( 프로그램과 데이터를 메인 메모리에 옮겨야 실행 가능 )

 

 

메인메모리 ( 주기억 장치 , 1차 기억장치 )

 

프로세서 외부에 있으며 프로세서에서 즉각적 수행할 프로그램과 데이터를 저장하거나 프로세서에서 처리한 결과를 

저장

 

- 다수의 셀로 구성

- 프로세서와 보조기억장치 사이에 있음

- 여기서 발생하는 디스크 입출력 병목 현상을 해결하기도 함

병목(bottleneck) 현상은 전체 시스템의 성능이나 용량이 하나의 구성 요소로 인해 제한을 받는 현상

 

 

 

물리적 주소 : 컴퓨터에 주어진 주소 ( 프로그래머는 물리적 주소 대신 수식이나 변수 사용 )

논리적 주소 : 컴파일러가 프로그램을 기계 명령어로 변환할때 변수와 명령어에 할당한 주소

 

매핑 , 메모리 맵 :  논리적 주소를 물리적 주소로 변환하는 과정

 

메모리 속도는 메모리 접근시간과 메모리 사이클 시간으로 표현할 수 있다.

 

메모리 접근 시간 : 명령이 발생한 후 목표 주소를 검색해 데이터 쓰기(읽기)를 시작할 때까지 걸린 시간

메모리 사이클 시간 : 두 번 연속적인 메모리 동작 사이에 필요한 최소 지연시간

 

캐시
 

프로세서 내부나 외부에 있으며, 처리 속도가 빠른 프로세서와 속도가 느린 메인 메모리의 속도 차이를 보완하는 고속 버퍼

 

- 메인 메모리에서 데이터를 블록 단위로 가져와 프로세서에 워드 단위로 전달하여 속도를 높인다.

- 데이터가 이동하는 통로 ( 대역폭 )을 확대하여 프로세서와 메모리의 속도 차이를 줄인다.

 

 

공간적 지역성 : 대부분의 프로그램이 참조한 주소와 인접한 주소의 내용을 다시 참조하는 특성

시간적 지역성 : 한 번 참조한 주소를 곧 다시 참조한다는 특성

 

보조 기억 장치 ( 2차 기억장치 , 외부기억장치 )

 

주변 장치 중 프로그램과 데이터를 저장하는 하드웨어

- 자기디스크 , 광디스크 , 자기테이프

 

시스템 버스

하드웨어를 물리적으로 연결하여 서로 데아터를 주고 받을 수 있게 하는 통로

 

 

데이터 버스 : 프로세서와 메인 메모리 , 주변 장치 사이에서 데이터를 전송

주소 버스 : 프로세서가 시스템 구성 요소를 식별하는 주소 정보 전송

제어 버스 : 프로세서가 시스템의 구성 요소를 제어 하는데 사용

 

 

 

주변장치

프로세서와 메인 메모리를 제외한 나머지 하드웨어 구성 요소
